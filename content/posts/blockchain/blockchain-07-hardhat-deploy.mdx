지난 편에서 Remix로 컨트랙트를 작성하고 실행해봤다. Remix는 빠르게 실험할 때 좋지만, 실제 프로젝트에는 부족하다. 테스트 자동화, CI/CD 연동, 팀 협업을 위해 로컬 개발 환경이 필요하다. 그게 **Hardhat**이다.

---

## Hardhat이 뭔가

Hardhat은 이더리움 스마트 컨트랙트 개발 프레임워크다. 비슷한 도구로 Foundry가 있는데, Foundry는 Solidity로 테스트를 작성하고 속도가 빠른 반면, Hardhat은 JavaScript/TypeScript로 테스트와 스크립트를 작성한다. 프론트엔드 개발자에게는 Hardhat이 진입 장벽이 낮다.

Hardhat이 제공하는 것:
- **Hardhat Network**: 로컬 이더리움 노드 (테스트용)
- **컴파일러**: Solidity → EVM 바이트코드 + ABI 생성
- **테스트 러너**: Mocha + Chai 기반
- **스크립트 실행**: 배포 자동화
- **플러그인 시스템**: Etherscan 검증, gas reporter 등

## 프로젝트 세팅

```bash
mkdir my-contract && cd my-contract
npm init -y
npm install --save-dev hardhat
npx hardhat init
```

`npx hardhat init`을 실행하면 프로젝트 타입을 선택하는 인터랙티브 메뉴가 나온다.

```
? What do you want to do?
❯ Create a JavaScript project
  Create a TypeScript project
  Create a TypeScript project (with Viem)
  Create an empty hardhat.config.js
  Quit
```

"Create a JavaScript project"를 선택하면 샘플 컨트랙트, 테스트, 배포 스크립트, 설정 파일이 자동으로 생성된다.

```
my-contract/
├── contracts/
│   └── Lock.sol          # 샘플 컨트랙트 (나중에 삭제)
├── scripts/
│   └── deploy.js         # 샘플 배포 스크립트
├── test/
│   └── Lock.js           # 샘플 테스트
├── hardhat.config.js     # 설정 파일
├── package.json
└── .gitignore
```

`contracts/Counter.sol`에 지난 편의 카운터 컨트랙트를 저장한다.

## 컴파일

```bash
npx hardhat compile
```

컴파일이 성공하면 `artifacts/` 폴더가 생성된다.

```
artifacts/
└── contracts/
    └── Counter.sol/
        └── Counter.json  # ABI + 바이트코드
```

`Counter.json`의 내용:

```json
{
  "contractName": "Counter",
  "abi": [
    {
      "name": "increment",
      "type": "function",
      "inputs": [],
      "outputs": [],
      "stateMutability": "nonpayable"
    },
    {
      "name": "count",
      "type": "function",
      "inputs": [],
      "outputs": [{ "name": "", "type": "uint256" }],
      "stateMutability": "view"
    }
    // ...
  ],
  "bytecode": "0x6080604052..."
```

ABI는 프론트엔드에서 컨트랙트와 통신할 때 쓴다. 바이트코드는 배포 시 블록체인에 올라가는 실제 코드다.

## 배포 스크립트

`scripts/deploy.js`:

```javascript
const hre = require("hardhat")

async function main() {
  // 배포 계정 정보
  const [deployer] = await hre.ethers.getSigners()
  console.log("배포 계정:", deployer.address)
  console.log("잔액:", hre.ethers.formatEther(
    await hre.ethers.provider.getBalance(deployer.address)
  ), "ETH")

  // 컨트랙트 팩토리 가져오기
  const Counter = await hre.ethers.getContractFactory("Counter")

  // 배포 (트랜잭션 전송)
  console.log("배포 중...")
  const counter = await Counter.deploy()

  // 배포 완료까지 대기
  await counter.waitForDeployment()

  const address = await counter.getAddress()
  console.log("Counter 배포 주소:", address)

  // 초기 상태 확인
  const initialCount = await counter.count()
  console.log("초기 count:", initialCount.toString())
}

main()
  .then(() => process.exit(0))
  .catch((error) => {
    console.error(error)
    process.exit(1)
  })
```

로컬 네트워크에 배포:

```bash
# 별도 터미널에서 npx hardhat node가 실행 중이어야 함
npx hardhat run scripts/deploy.js --network localhost
```


## 테스트 작성

테스트는 Hardhat 내장 테스트 러너(Mocha + Chai)로 작성한다. `npx hardhat test`를 실행하면 별도로 노드를 띄울 필요 없이 자동으로 처리된다.

`test/Counter.test.js`:

```javascript
const { expect } = require("chai")
const { ethers } = require("hardhat")

describe("Counter", function () {
  // 각 테스트 전에 새 컨트랙트 배포
  async function deployCounterFixture() {
    const [owner, addr1, addr2] = await ethers.getSigners()
    const Counter = await ethers.getContractFactory("Counter")
    const counter = await Counter.deploy()
    return { counter, owner, addr1, addr2 }
  }

  describe("초기 상태", function () {
    it("count는 0으로 시작해야 한다", async function () {
      const { counter } = await deployCounterFixture()
      expect(await counter.count()).to.equal(0)
    })

    it("owner는 배포한 계정이어야 한다", async function () {
      const { counter, owner } = await deployCounterFixture()
      expect(await counter.owner()).to.equal(owner.address)
    })
  })

  describe("increment", function () {
    it("count가 1 증가해야 한다", async function () {
      const { counter } = await deployCounterFixture()
      await counter.increment()
      expect(await counter.count()).to.equal(1)
    })

    it("CountChanged 이벤트가 발생해야 한다", async function () {
      const { counter, owner } = await deployCounterFixture()
      await expect(counter.increment())
        .to.emit(counter, "CountChanged")
        .withArgs(owner.address, 0, 1)
    })

    it("누구나 호출할 수 있어야 한다", async function () {
      const { counter, addr1 } = await deployCounterFixture()
      await counter.connect(addr1).increment()
      expect(await counter.count()).to.equal(1)
    })
  })

  describe("reset", function () {
    it("owner만 reset할 수 있어야 한다", async function () {
      const { counter, addr1 } = await deployCounterFixture()
      await counter.increment()

      // 다른 계정으로 reset 시도 → 실패해야 함
      await expect(
        counter.connect(addr1).reset()
      ).to.be.revertedWith("Only owner")
    })

    it("owner가 reset하면 count가 0이 된다", async function () {
      const { counter } = await deployCounterFixture()
      await counter.increment()
      await counter.increment()
      await counter.reset()
      expect(await counter.count()).to.equal(0)
    })
  })
})
```

```bash
npx hardhat test
```

출력:
```
  Counter
    초기 상태
      ✔ count는 0으로 시작해야 한다
      ✔ owner는 배포한 계정이어야 한다
    increment
      ✔ count가 1 증가해야 한다
      ✔ CountChanged 이벤트가 발생해야 한다
      ✔ 누구나 호출할 수 있어야 한다
    reset
      ✔ owner만 reset할 수 있어야 한다
      ✔ owner가 reset하면 count가 0이 된다

  7 passing (1.2s)
```

## Sepolia 테스트넷에 배포

실제 테스트넷에 배포하려면:

**1. Sepolia ETH 받기**

sepoliafaucet.com 또는 alchemy.com/faucets 에서 Sepolia ETH를 무료로 받는다. MetaMask 주소를 입력하면 된다.

**2. Alchemy에서 RPC URL 받기**

alchemy.com에서 가입 → 새 앱 생성 → Sepolia 선택 → API Key 확인

**3. 환경변수 설정**

`.env` 파일 (절대 git에 커밋하지 말 것):

```
SEPOLIA_RPC_URL=https://eth-sepolia.g.alchemy.com/v2/YOUR_API_KEY
PRIVATE_KEY=0x메타마스크_개인키_내보내기로_가져옴
ETHERSCAN_API_KEY=etherscan.io에서_발급
```

`.gitignore`에 `.env` 추가:
```
.env
artifacts/
cache/
node_modules/
```

**4. hardhat.config.js 설정**

```javascript
require("@nomicfoundation/hardhat-toolbox")
require("dotenv").config()

module.exports = {
  solidity: "0.8.20",
  networks: {
    sepolia: {
      url: process.env.SEPOLIA_RPC_URL,
      accounts: [process.env.PRIVATE_KEY],
      chainId: 11155111,
    }
  },
  etherscan: {
    apiKey: process.env.ETHERSCAN_API_KEY
  }
}
```

**5. 배포**

```bash
npm install dotenv
npx hardhat run scripts/deploy.js --network sepolia
```

```

잔액: 0.2 ETH
배포 중...

```

## Etherscan에서 소스코드 검증

배포된 주소를 Sepolia Etherscan(sepolia.etherscan.io)에서 검색하면 컨트랙트를 볼 수 있다. 하지만 기본적으로는 바이트코드만 보인다.

소스코드를 공개하고 싶다면 검증(Verify)이 필요하다.

```bash
npx hardhat verify --network sepolia 0x배포된_주소
```

검증이 완료되면 Etherscan에서 Solidity 소스코드를 볼 수 있고, 웹 인터페이스로 함수를 직접 호출할 수도 있다.
