롤업에는 두 가지 방식이 있다고 했다. Optimistic Rollup과 ZK Rollup. 둘 다 L2에서 실행하고 L1에 데이터를 올린다. 그런데 L1이 "이 데이터가 맞다"를 검증하는 방식이 완전히 다르다.

---

## Optimistic Rollup: 일단 믿고, 이의 있으면 말해

Optimistic Rollup의 철학은 **낙관적 가정**이다.

> "L2 운영자(시퀀서)가 올바른 상태를 게시했다고 일단 가정한다. 틀렸다면 누군가 이의를 제기하겠지."

### 작동 방식

```
1. L2에서 트랜잭션 처리
2. 결과 상태(state root)를 L1에 게시
3. 7일의 이의 제기 기간(Challenge Period) 시작
4. 기간 내 아무도 이의 제기 없음 → 확정
   이의 제기 있음 → 분쟁 해결
```

### 사기 증명(Fraud Proof)

누군가 "시퀀서가 거짓 상태를 게시했다"고 판단하면 이의를 제기한다.

이의 제기자는 L1에 **사기 증명(Fraud Proof)**을 제출한다. L1 스마트 컨트랙트가 문제의 트랜잭션을 직접 재실행해서 검증한다. 만약 게시된 상태가 틀렸다면 시퀀서가 패널티를 받고 올바른 상태로 롤백된다.

이의 제기자는 맞으면 포상을 받고, 틀리면 자신이 맡긴 보증금을 잃는다. 이 인센티브 구조가 정직한 시스템을 유지하게 만든다.

### Optimistic Rollup 출금이 7일 걸리는 이유

L2 → L1으로 자산을 이동(출금)하려면 7일을 기다려야 한다.

왜? 시퀀서가 게시한 상태가 확정되기까지 이의 제기 기간이 필요하기 때문이다. 이 기간이 끝나기 전에 출금을 허용하면, 이후 사기 증명으로 상태가 롤백될 때 이미 빠져나간 자산을 되돌릴 방법이 없다.

빠른 출금을 원하면 **브릿지 유동성 공급자**를 통해 즉시 받는 대신 소정의 수수료를 낸다. 공급자가 먼저 L1 자산을 내주고, 7일 후에 L2에서 자산을 회수하는 구조다.

## ZK Rollup: 수학으로 증명한다

ZK Rollup의 철학은 반대다.

> "상태가 올바르다는 수학적 증명을 함께 제출한다. 검증은 L1에서 즉시."

### 작동 방식

```
1. L2에서 트랜잭션 처리
2. 수백~수천 개 트랜잭션의 실행이 올바르다는 ZK 증명(proof) 생성
   (증명 생성에 수초~수십초 소요)
3. proof와 결과 상태를 L1에 함께 게시
4. L1 컨트랙트가 proof를 검증 (수밀리초)
5. 검증 성공 → 즉시 확정
```

이의 제기 기간이 없다. 수학적으로 올바르다는 게 증명됐으니까.

그래서 ZK Rollup 출금은 몇 분 내로 완료된다.

---

## ZK 증명이란 — 직관적으로 이해하기

**ZK(Zero-Knowledge, 영지식)** 증명은 "내가 어떤 정보를 알고 있다는 걸 그 정보를 공개하지 않고 증명하는 방법"이다.

블록체인에서 쓰이는 ZK 증명의 역할은 조금 다르다. "이 계산이 올바르게 수행됐다는 걸 계산 전체를 다시 실행하지 않고 증명하는 것"이다.

비유: 술래잡기를 생각해보자. 내가 숨어있다는 걸 증명하려면 원래는 나와야 한다. 하지만 ZK 증명은 "나는 숨어있다, 그리고 숨어있는 위치를 네가 알 필요 없이 이 사실만 믿어도 된다는 수학적 증거를 제출한다."

수천 개 트랜잭션을 L1에서 다시 실행하지 않아도, 작은 proof 하나로 "이 트랜잭션들이 모두 올바르게 처리됐다"는 걸 검증할 수 있다.

## zk-SNARK vs zk-STARK

ZK 증명에는 두 가지 주요 방식이 있다.

### zk-SNARK
(Zero-Knowledge Succinct Non-interactive Argument of Knowledge)

특징:
- 증명 크기가 매우 작다 (수백 바이트)
- L1 검증 비용이 저렴하다
- 신뢰 설정(Trusted Setup)이 필요하다

**신뢰 설정**이 단점이다. 시스템을 최초로 설정할 때 "독성 폐기물(toxic waste)"이라 불리는 비밀값이 생성된다. 이 비밀값이 노출되면 가짜 증명을 만들 수 있다. 설정 과정에 참여한 모든 참여자가 이 비밀값을 파기했다고 믿어야 한다.

zkSync, Polygon zkEVM 등이 zk-SNARK 계열을 사용한다.

### zk-STARK
(Zero-Knowledge Scalable Transparent Argument of Knowledge)

특징:
- 신뢰 설정이 필요 없다 (Transparent)
- 양자 컴퓨터에 더 안전하다
- 증명 크기가 zk-SNARK보다 크다
- L1 검증 비용이 상대적으로 비싸다

StarkNet이 zk-STARK를 사용한다.

| | zk-SNARK | zk-STARK |
|--|---------|---------|
| 증명 크기 | 작음 (~200 bytes) | 큼 (~10-100 KB) |
| 검증 속도 | 빠름 | 느림 |
| 신뢰 설정 | 필요 | 불필요 |
| 양자 내성 | 없음 | 있음 |
| 대표 프로젝트 | zkSync, Polygon | StarkNet |

## Optimistic vs ZK: 언제 무엇을 선택하나

| | Optimistic Rollup | ZK Rollup |
|--|-----------------|---------|
| 출금 대기 | 7일 | 수분~수시간 |
| EVM 호환성 | 완벽 | 제한적~점차 개선 |
| 증명 비용 | 없음 | ZK 증명 생성 비용 |
| 보안 전제 | 1명 이상의 정직한 감시자 | 수학적 증명 |
| 대표 체인 | Arbitrum, Optimism | zkSync, StarkNet |

EVM 호환성이 Optimistic의 강점이었다. 기존 이더리움 스마트 컨트랙트를 거의 수정 없이 배포할 수 있다.

ZK Rollup은 EVM 실행을 ZK 증명으로 표현하는 게 기술적으로 어려웠다. 최근 몇 년간 zkEVM이 등장하면서 이 격차가 줄어들고 있다.
