이더리움의 가스비가 비싼 날이 있다. NFT 민팅 러시나 DeFi 이벤트 때 가스비가 수십~수백 달러까지 오르는 걸 본 적이 있을 것이다.

왜 이런 일이 벌어질까? 이더리움 메인넷은 왜 더 많은 트랜잭션을 처리하도록 확장하지 않는 걸까?

그 답이 **확장성 트릴레마**다.

---

## 왜 이더리움 메인넷은 느린가

이더리움 메인넷은 현재 초당 약 15~30개의 트랜잭션을 처리한다(TPS). VISA 카드는 초당 약 24,000 TPS를 처리한다. 차이가 엄청나다.

왜 이렇게 느릴까?

기억하자. 이더리움의 모든 노드는 모든 트랜잭션을 직접 검증한다. 수천 개의 노드가 모두 같은 연산을 반복하는 것이다.

블록 생성 주기가 12초이고, 블록 크기(가스 한도)가 제한돼 있다. 블록이 더 크면 더 많은 트랜잭션이 담기지만, 더 큰 블록은 처리하는 데 더 오래 걸린다. 처리 시간이 12초를 넘어버리면 네트워크 안정성이 무너진다.

그렇다면 블록 크기를 키우고, 블록 생성 주기를 줄이면 되지 않나? 여기서 트릴레마가 등장한다.

---

## 확장성 트릴레마

Vitalik Buterin이 제시한 개념이다. 블록체인은 다음 세 가지를 동시에 완벽히 달성할 수 없다.

```
        탈중앙화
       /         \
      /           \
  보안  ----- 확장성
```

**탈중앙화**: 누구나 노드를 운영해서 검증에 참여할 수 있어야 한다.

**보안**: 공격하거나 조작하기 극도로 어려워야 한다.

**확장성**: 많은 트랜잭션을 빠르고 저렴하게 처리할 수 있어야 한다.

셋 중 두 가지를 선택하면 나머지 하나를 포기해야 한다.

### 확장성을 높이면 탈중앙화가 무너진다

블록을 아주 크게 만들면 많은 트랜잭션이 담겨 TPS가 오른다. 그런데 큰 블록을 처리하려면 빠른 인터넷과 강력한 하드웨어가 필요하다. 일반인은 노드를 운영하기 어려워진다. 소수의 대형 서버만 남는다.

BSC(BNB Smart Chain)가 이 방향이다. 이더리움보다 훨씬 빠르지만, 검증자가 21개뿐이다. 탈중앙화를 상당 부분 포기했다.

### 확장성을 높이면 보안이 약해진다

블록 생성 주기를 너무 줄이면, 블록이 전 세계 노드에 전파되기 전에 다음 블록이 생성될 수 있다. 이러면 포크가 자주 발생하고 체인이 불안정해진다.

### 이더리움의 선택

이더리움은 **탈중앙화 + 보안**을 선택했다. 확장성은 L2에 위임한다.

## 롤업이 트릴레마를 우회하는 방법

롤업의 핵심 아이디어:

> 트랜잭션 실행은 L2에서 빠르게. 단, 데이터와 보안은 L1(이더리움)을 활용.

구체적으로:

```
[L2 환경]
1. 사용자들이 빠르고 저렴하게 L2에서 트랜잭션 처리
2. 수천 개의 트랜잭션을 하나의 묶음(배치)으로 압축

[L1에 기록]
3. 압축된 배치를 이더리움 메인넷에 게시
4. 이더리움이 해당 데이터를 영구 보존
```

이더리움은 실행을 직접 하지 않는다. 데이터를 보존하고, 분쟁 발생 시 최종 판단을 내리는 역할만 한다.

그 결과:
- **속도**: L2에서 처리하므로 수천 TPS 가능
- **비용**: 수천 개 트랜잭션이 L1 비용을 나눠 부담
- **보안**: 최종 데이터가 이더리움에 있어서 이더리움 보안 수준을 상속

---

## 데이터 가용성이 핵심이다

롤업이 보안을 L1에서 상속받으려면 **트랜잭션 데이터가 L1에 공개적으로 올라와 있어야 한다.**

왜? 누군가 L2 운영자가 거짓 상태를 주장할 때, 다른 사람이 L1의 데이터를 보고 이의를 제기하거나 올바른 상태를 재계산할 수 있어야 하기 때문이다.

데이터가 L1에 없다면 — 이의 제기가 불가능하다. L2 운영자를 그냥 믿어야 한다. 그건 중앙화다.

롤업이 L1에 게시하는 데이터를 **calldata** 또는 **Blob**(EIP-4844 이후)라고 부른다.

---

## EIP-4844 이전과 이후: L2 비용이 크게 달라진 이유

2024년 초 Cancun 업그레이드에서 도입된 **EIP-4844(Proto-Danksharding)**은 L2 수수료를 획기적으로 낮췄다.

이전: L2는 데이터를 calldata로 L1에 올렸다. Calldata는 영구적으로 블록에 저장되어 모든 풀 노드가 평생 갖고 있어야 했다. 비쌌다.

이후: **Blob** 트랜잭션이 도입됐다. Blob은 약 125KB 크기의 데이터 덩어리로, 약 18일 후 자동 삭제된다. 롤업 검증에 필요한 기간만 보존되고 이후엔 삭제해도 된다. 영구 저장이 아니므로 훨씬 저렴하다.

Blob 도입 후 Arbitrum, Optimism 같은 L2의 트랜잭션 수수료가 평균 10배 이상 저렴해졌다.

## L2 생태계 현황

2026년 기준으로 주요 L2들의 TVL(Total Value Locked)은 수십억 달러 수준이다.

| L2 | 방식 | 주요 특징 |
|----|------|---------|
| Arbitrum | Optimistic Rollup | 가장 큰 TVL |
| Optimism | Optimistic Rollup | OP 스택, Superchain |
| zkSync | ZK Rollup | EVM 호환 ZK |
| StarkNet | ZK Rollup | STARK 증명, Cairo 언어 |
| Base | Optimistic Rollup | Coinbase 운영, OP 스택 |
| Polygon zkEVM | ZK Rollup | 폴리곤 재단 |

---

다음 편에서는 Optimistic Rollup과 ZK Rollup의 내부 메커니즘 차이를 다룬다. 사기 증명과 유효성 증명이 각각 어떻게 작동하는지, zk-SNARK와 zk-STARK의 직관적 차이가 무엇인지를 살펴본다.
