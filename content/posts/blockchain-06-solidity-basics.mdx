지난 편에서 스마트 컨트랙트는 Solidity로 작성한다고 했다. 이번 편에서는 Solidity 문법을 JavaScript와 비교하면서 실제로 읽고 쓸 수 있는 수준까지 다룬다.

Solidity는 배워야 할 새 언어지만, JavaScript를 알고 있다면 기본 구조는 금방 익힌다. 다만 블록체인 특유의 개념들이 섞여 있어서 그 부분에 집중해야 한다.

---

## 파일 구조와 컨트랙트 선언

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.20;

contract MyContract {
    // 상태 변수, 이벤트, 함수...
}
```

- `// SPDX-License-Identifier`: 소스 코드 라이선스. 없으면 컴파일 경고가 뜬다. MIT가 가장 많이 쓰인다.
- `pragma solidity ^0.8.20`: 이 버전 이상의 컴파일러를 쓰라는 선언. `^`는 npm semver와 같다.
- `contract`: JavaScript의 `class`와 비슷하다. 상태 변수와 함수를 묶는 단위다.

---

## 변수: 타입 명시 필수

JavaScript는 `let x = 1`로 끝이지만, Solidity는 타입을 반드시 명시해야 한다.

```solidity
// 정수
uint256 public count;      // 부호 없는 정수, 0 ~ 2^256-1
int256 public temperature; // 부호 있는 정수, 음수 가능
uint8 public smallNum;     // 8비트, 0~255 (가스 절약용으로 쓰기도 함)

// 논리
bool public isActive;      // true / false

// 주소
address public owner;      // 이더리움 주소 (20바이트)
address payable public recipient; // ETH를 보낼 수 있는 주소 타입

// 문자열 & 바이트
string public name;        // 가변 길이 문자열 (가스 비쌈)
bytes32 public hash;       // 고정 32바이트 (문자열보다 가스 저렴)
bytes public data;         // 가변 길이 바이트 배열
```

**가스 관점에서의 선택**: 문자열 데이터를 꼭 온체인에 저장해야 한다면 `bytes32`가 `string`보다 효율적이다. 32바이트 이하 데이터는 `bytes32`로 처리하는 게 일반적이다.

**상태 변수 vs 지역 변수**

```solidity
contract Example {
    uint256 public storedValue; // 상태 변수: 블록체인에 영구 저장

    function calculate(uint256 x) public pure returns (uint256) {
        uint256 result = x * 2; // 지역 변수: 함수 실행 중에만 메모리에 존재
        return result;
    }
}
```

상태 변수는 블록체인 스토리지에 저장된다. 읽고 쓰는 데 가스가 가장 많이 든다. 지역 변수는 EVM 메모리에만 저장되며 함수 실행 후 사라진다.

## 함수 제어자

Solidity 함수에는 여러 제어자가 붙는다. 처음엔 많아 보이지만 패턴이 있다.

```solidity
// 가시성 (Visibility)
function pub() public {}       // 누구나 호출 가능
function ext() external {}     // 외부에서만 호출 가능 (내부 호출 불가)
function intern() internal {}  // 이 컨트랙트 + 상속 컨트랙트만
function priv() private {}     // 이 컨트랙트 내부만

// 상태 접근 (State Mutability)
function read() public view returns (uint256) {
    return count; // 상태 읽기만 → 가스 없음
}

function calc(uint256 x) public pure returns (uint256) {
    return x * 2; // 상태 접근 없음 → 가스 없음
}

function write(uint256 val) public {
    count = val; // 상태 변경 → 가스 필요
}

function sendETH() public payable {
    // ETH를 받을 수 있는 함수 → payable 필요
    deposits[msg.sender] += msg.value;
}
```

**프론트엔드 관점에서의 핵심**:
- `view` / `pure` 함수 → 그냥 읽기, 가스비 없음
- 나머지 → 트랜잭션, MetaMask 서명 필요, 가스비 있음

## 전역 변수

Solidity에는 현재 트랜잭션과 블록에 대한 정보를 담은 전역 변수가 있다.

```solidity
function example() public payable {
    address caller = msg.sender;   // 이 함수를 호출한 주소
    uint256 value = msg.value;     // 함께 보낸 ETH (Wei 단위)
    bytes memory data = msg.data;  // 호출 데이터 (calldata)

    uint256 blockNum = block.number;    // 현재 블록 번호
    uint256 timestamp = block.timestamp; // 현재 블록 타임스탬프 (Unix)
    address miner = block.coinbase;     // 이 블록을 검증한 주소
}
```

`msg.sender`와 `msg.value`는 거의 모든 컨트랙트에서 쓰인다. 오너 권한 체크, 입금 처리에 필수다.

## 에러 처리: require, revert, assert

```solidity
function withdraw(uint256 amount) public {
    // require: 조건 불충족 시 실행 취소 + 에러 메시지
    require(msg.sender == owner, "Only owner can withdraw");
    require(balance >= amount, "Insufficient balance");

    balance -= amount;
    payable(msg.sender).transfer(amount);
}

function criticalOperation() public {
    // revert: 명시적으로 실행 취소 (조건 없이 중단 가능)
    if (someComplexCondition()) {
        revert("Complex condition failed");
    }
}

function internalCheck(uint256 x) internal pure {
    // assert: 절대 false가 되면 안 되는 불변식 검증 (내부 버그 감지용)
    assert(x > 0); // 이게 false면 코드에 버그가 있는 것
}
```

**언제 무엇을 쓰나**:
- `require`: 외부 입력 검증, 조건 불충족 시 가스 일부 환불
- `revert`: 복잡한 조건 분기, `require`와 동작 같음
- `assert`: 내부 로직 버그 탐지용, 실패 시 가스 전액 소비 (더 엄격)

## mapping, array, struct

**mapping**

JavaScript의 `Map` 또는 객체와 비슷하지만, 모든 키에 대해 기본값이 존재하고 iterate(순회)가 불가능하다.

```solidity
mapping(address => uint256) public balances;
mapping(address => mapping(address => uint256)) public allowances; // 중첩 mapping

// 읽기/쓰기
balances[msg.sender] = 100;
uint256 myBalance = balances[msg.sender];

// 존재 여부 확인
// mapping은 항상 기본값(0, false, address(0))을 반환하므로
// "키가 없다"와 "값이 0이다"를 구분하기 어렵다
// → 별도의 bool mapping을 함께 쓰거나 struct를 활용
```

**array**

```solidity
uint256[] public dynamicArray;  // 가변 크기
uint256[10] public fixedArray;  // 고정 크기 (가스 더 효율적)

dynamicArray.push(42);
dynamicArray.push(99);
uint256 first = dynamicArray[0];  // 42
uint256 len = dynamicArray.length; // 2

// 배열 삭제는 주의 필요
// delete dynamicArray[1]은 해당 인덱스를 0으로 만들 뿐 길이는 변하지 않음
// 실제로 제거하려면 마지막 요소와 교체 후 pop()
dynamicArray[1] = dynamicArray[dynamicArray.length - 1];
dynamicArray.pop();
```

**struct**

```solidity
struct Proposal {
    string description;
    uint256 voteCount;
    bool executed;
    address proposer;
}

Proposal[] public proposals;

function createProposal(string memory desc) public {
    proposals.push(Proposal({
        description: desc,
        voteCount: 0,
        executed: false,
        proposer: msg.sender
    }));
}

// mapping과 조합
mapping(uint256 => Proposal) public proposalById;
mapping(address => bool) public hasVoted;
```

## 이벤트 (Events)

이벤트는 블록체인에 로그를 남긴다. 스토리지보다 훨씬 가스가 저렴하고, 프론트엔드에서 구독할 수 있다.

```solidity
// 이벤트 선언
event Transfer(
    address indexed from,  // indexed: 이 값으로 필터링 가능
    address indexed to,
    uint256 amount         // indexed 없으면 필터링 불가
);

event Approval(address indexed owner, address indexed spender, uint256 value);

// 이벤트 발생
function transfer(address to, uint256 amount) public {
    require(balances[msg.sender] >= amount, "Insufficient balance");
    balances[msg.sender] -= amount;
    balances[to] += amount;

    emit Transfer(msg.sender, to, amount);
}
```

`indexed` 키워드가 붙은 파라미터는 이벤트 로그의 "토픽"으로 분류돼 필터링이 가능하다. 최대 3개까지 `indexed`를 붙일 수 있다.

Ethers.js에서 이벤트를 구독하면:
```javascript
contract.on("Transfer", (from, to, amount) => {
    console.log(`${from} → ${to}: ${amount}`)
})
```

## modifier: 재사용 가능한 조건 체크

동일한 조건 검사가 여러 함수에 반복된다면 `modifier`로 추출할 수 있다.

```solidity
contract Ownable {
    address public owner;

    constructor() {
        owner = msg.sender;
    }

    // modifier 선언
    modifier onlyOwner() {
        require(msg.sender == owner, "Not the owner");
        _; // 이 위치에 실제 함수 본문이 들어간다
    }

    modifier nonZeroAmount(uint256 amount) {
        require(amount > 0, "Amount must be positive");
        _;
    }

    // modifier 사용
    function setOwner(address newOwner) public onlyOwner {
        owner = newOwner;
    }

    function withdraw(uint256 amount) public onlyOwner nonZeroAmount(amount) {
        payable(owner).transfer(amount);
    }
}
```

`_`는 modifier 내에서 "함수 본문을 여기 실행"하라는 자리 표시자다. `_` 전에 코드를 쓰면 함수 실행 전, 후에 쓰면 함수 실행 후에 실행된다.

## 실습: 카운터 컨트랙트

Remix IDE(remix.ethereum.org)에서 바로 실행할 수 있다. 설치 없이 브라우저에서 Solidity를 작성하고 배포할 수 있다.

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.20;

contract Counter {
    uint256 public count;
    address public owner;
    uint256 public constant MAX_COUNT = 1000; // 상수 (대문자 관례)

    event CountChanged(address indexed by, uint256 oldCount, uint256 newCount);

    modifier onlyOwner() {
        require(msg.sender == owner, "Only owner");
        _;
    }

    modifier withinBounds() {
        require(count < MAX_COUNT, "Already at max");
        _;
    }

    constructor() {
        owner = msg.sender;
        count = 0;
    }

    function increment() public withinBounds {
        uint256 old = count;
        count += 1;
        emit CountChanged(msg.sender, old, count);
    }

    function decrement() public {
        require(count > 0, "Already at zero");
        uint256 old = count;
        count -= 1;
        emit CountChanged(msg.sender, old, count);
    }

    function incrementBy(uint256 amount) public withinBounds {
        uint256 old = count;
        count += amount;
        if (count > MAX_COUNT) count = MAX_COUNT; // 최대치 초과 방지
        emit CountChanged(msg.sender, old, count);
    }

    function reset() public onlyOwner {
        uint256 old = count;
        count = 0;
        emit CountChanged(msg.sender, old, 0);
    }
}
```

**Remix 실습 순서**:

1. remix.ethereum.org 접속
2. 좌측 파일 탐색기 → 새 파일 `Counter.sol` 생성
3. 위 코드 붙여넣기
4. 좌측 **Solidity Compiler** → Compile Counter.sol
5. 좌측 **Deploy & Run Transactions** → Environment: Remix VM → Deploy
6. 배포된 컨트랙트 펼치기 → `increment`, `decrement`, `reset` 버튼 클릭

`increment`를 클릭할 때마다 `count`가 올라간다. `reset`은 처음 배포한 계정(Account 드롭다운의 첫 번째)에서만 실행된다. 다른 계정으로 바꿔서 `reset`을 호출하면 "Only owner" 에러가 뜬다.
