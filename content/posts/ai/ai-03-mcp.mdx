지금까지는 AI와 대화하는 방식을 다뤘다. 이번 편부터는 다르다. AI에게 **도구를 직접 연결**하는 방법이다.

---

## 프롬프트로 설명하는 것의 한계

지금까지 AI를 쓰는 방식은 대부분 이렇다.

```
src/components/Button.tsx 파일을 보면 이런 구조야:
[코드 복붙]

여기에 loading 상태를 추가해줘.
```

AI에게 파일 내용을 직접 붙여넣어야 한다. 파일이 여러 개면 전부 복사해서 넣어야 한다. 파일이 바뀌면 다시 복사해야 한다.

MCP는 이 방식을 바꾼다.

---

## MCP가 뭔가

**MCP(Model Context Protocol)**는 AI가 외부 도구를 직접 호출할 수 있게 해주는 표준 프로토콜이다. Anthropic이 2024년 말에 공개했고, 현재 Claude, GPT-4o, Gemini 등 주요 AI가 지원한다.

쉽게 말하면 이렇다.

기존: 나 → 파일 복사 → AI에게 붙여넣기 → AI가 읽음
MCP: AI → MCP 서버에 직접 요청 → 파일 읽음

AI가 직접 도구를 호출하기 때문에, 내가 중간에서 복사/붙여넣기를 할 필요가 없다.

---

## 어떻게 작동하는가

MCP는 클라이언트-서버 구조다.

```
[AI 클라이언트] ←→ [MCP 서버] ←→ [실제 도구/서비스]
 Claude Code          filesystem       로컬 파일 시스템
 Cursor               browser          웹 브라우저
 Windsurf             github           GitHub 레포
                      postgresql       DB
```

AI 클라이언트(Claude Code, Cursor 등)가 MCP 서버에 요청을 보낸다. MCP 서버는 실제 도구를 호출하고 결과를 AI에게 돌려준다.

AI 입장에서는 "파일 읽기", "웹 검색", "DB 조회" 같은 도구가 생기는 것이다.

---

## 실제로 뭐가 달라지는가

MCP 없이:
```
나: 이 컴포넌트에서 에러가 나. [코드 200줄 복붙]
AI: 이런 부분이 문제인 것 같습니다...
```

MCP (filesystem) 있이:
```
나: src/components/Button.tsx에서 에러가 나.
AI: [파일을 직접 읽음] 47번째 줄 onClick 핸들러에서 타입 에러가 납니다.
    [파일을 직접 수정함]
```

파일 경로만 알려주면 AI가 직접 열고, 읽고, 수정한다. 내가 복사/붙여넣기를 할 필요가 없다.

다른 예시:

```
나: caniuse에서 CSS subgrid 지원 브라우저 확인해줘.
AI: [브라우저 MCP로 caniuse.com 직접 접속]
    현재 Chrome 117+, Firefox 71+, Safari 16+ 지원합니다.
```

```
나: users 테이블에서 오늘 가입한 사람 수 알려줘.
AI: [DB MCP로 직접 쿼리]
    오늘 신규 가입자: 143명
```

---

## MCP를 지원하는 클라이언트

현재 MCP를 지원하는 주요 도구들:

| 도구 | 특징 |
|------|------|
| **Claude Code** | Anthropic 공식 CLI. MCP 지원 가장 성숙 |
| **Cursor** | VS Code 기반 AI 에디터. 설정 UI 제공 |
| **Windsurf** | Codeium의 AI 에디터 |
| **Claude Desktop** | 데스크탑 앱. 로컬 MCP 서버 연결 가능 |
| **Continue** | VS Code / JetBrains 오픈소스 플러그인 |

ChatGPT는 자체 Tools/Function calling 방식을 쓰고 MCP를 직접 지원하지 않는다. 하지만 비슷한 역할을 하는 기능이 있다 — 다음 편에서 다룬다.

---

## 어떤 MCP 서버가 있는가

공개된 MCP 서버 중 자주 쓰이는 것들:

**파일/코드:**
- `filesystem` — 로컬 파일 읽기/쓰기/검색
- `git` — git log, diff, blame 조회

**웹/검색:**
- `fetch` — 웹 페이지 직접 접속
- `brave-search` / `tavily` — 웹 검색

**데이터:**
- `postgresql` / `sqlite` — DB 직접 쿼리
- `google-sheets` — 스프레드시트 읽기/쓰기

**개발 도구:**
- `github` — 이슈, PR, 코드 조회
- `linear` — 작업 관리
- `slack` — 메시지 전송/조회

---

## MCP의 보안 주의점

MCP는 강력한 만큼 주의가 필요하다.

AI가 파일을 직접 수정하고, DB에 직접 쿼리를 날리고, 외부 서비스에 요청을 보낼 수 있다. 잘못 설정하면 의도치 않은 파일이 수정되거나 데이터가 삭제될 수 있다.

기본 원칙:
- 필요한 MCP 서버만 연결 (전부 연결하지 않기)
- 파일 시스템 MCP는 접근 경로를 제한 (전체 `/` 말고 프로젝트 폴더만)
- DB MCP는 읽기 전용으로 시작하는 게 안전
- 프로덕션 DB보다 개발 DB로 먼저 테스트
