지난 편에서 MCP가 무엇인지 다뤘다. 이번 편은 실제로 연결하고 쓰는 방법이다.

---

## MCP를 연결하면 실제로 어떻게 쓰는가

설정 방법 전에 먼저 이걸 보자. MCP를 연결하면 **대화 중에 특별한 명령 없이 그냥 말하면 된다.** Claude가 알아서 도구를 쓴다.

filesystem MCP를 연결했다면:

```
나: src/components/Navbar.tsx 어떻게 생겼어?
Claude: [파일을 직접 읽음]
        Navbar 컴포넌트는 'use client'로 선언되어 있고,
        usePathname으로 현재 경로를 감지해서...
```

파일 내용을 복사해서 붙여넣지 않았다. Claude가 직접 읽은 것이다.

```
나: src/app 아래 파일 구조 전체 보여줘.
Claude: [디렉터리를 직접 탐색]
        src/app/
        ├── layout.tsx
        ├── page.tsx
        └── [locale]/
            ├── layout.tsx
            └── page.tsx
```

이런 식으로 동작한다. 설정만 해두면 그 다음은 그냥 대화하면 된다.

---

## Claude Code에서 MCP 설정

두 가지 방법이 있다.

**방법 1: 명령어로 추가 (전역 적용)**

```bash
claude mcp add filesystem -- npx -y @modelcontextprotocol/server-filesystem ~/projects
```

이후 모든 프로젝트에서 filesystem MCP를 쓸 수 있다.

**방법 2: .mcp.json 파일 (프로젝트별)**

프로젝트 루트에 `.mcp.json` 파일을 만든다. git에 함께 커밋하면 팀원도 같은 MCP 환경을 쓸 수 있다.

```json
{
  "mcpServers": {
    "filesystem": {
      "command": "npx",
      "args": [
        "-y",
        "@modelcontextprotocol/server-filesystem",
        "/Users/yourname/Desktop/project/my-portfolio"
      ]
    }
  }
}
```

설정 후 연결 확인:
```bash
claude mcp list
# filesystem: connected
```

## Cursor에서 MCP 설정

`~/.cursor/mcp.json` 파일을 만든다:

```json
{
  "mcpServers": {
    "filesystem": {
      "command": "npx",

        "-y",

        "/Users/yourname/projects"
      ]
    }
  }
}
```

또는 `Cmd + ,` → Settings → 검색창에 "MCP" → Add Server로 UI에서 추가.

## 자주 쓰는 MCP 서버 설정

### filesystem — 파일 직접 접근

```json
"filesystem": {
  "command": "npx",
  "args": ["-y", "@modelcontextprotocol/server-filesystem", "./src"]
}
```

접근 경로를 `./src`처럼 프로젝트 하위로 제한하는 게 안전하다.

### fetch — 웹 페이지 접근

```json
"fetch": {
  "command": "uvx",
  "args": ["mcp-server-fetch"]
}
```

`uvx`가 없으면 `pip install uv` 먼저. 이후 "caniuse에서 CSS grid 지원 현황 알려줘"처럼 URL 없이 바로 요청 가능.

### github — GitHub 레포 조회

```json
"github": {
  "command": "npx",
  "args": ["-y", "@modelcontextprotocol/server-github"],
  "env": {
    "GITHUB_PERSONAL_ACCESS_TOKEN": "ghp_your_token"
  }
}
```

토큰은 GitHub → Settings → Developer settings → Personal access tokens에서 발급.

연결 후 이런 요청이 가능해진다:
```
나: owner/repo 레포 최근 이슈 목록 보여줘.
나: PR #123 변경사항 요약해줘.
```

### postgresql — DB 직접 쿼리

```json
"postgresql": {
  "command": "npx",
  "args": ["-y", "@modelcontextprotocol/server-postgres", "postgresql://localhost/mydb"]
}
```

연결 후:
```
나: users 테이블 스키마 보여줘.
나: 오늘 가입한 사용자 수 알려줘.
```

처음엔 읽기 전용 DB나 개발 DB로 시작하는 걸 권장.

## 커스텀 MCP 서버 만들기 — 처음부터 끝까지

공개된 서버가 없는 경우, 직접 만들 수 있다. 회사 내부 API, 사내 검색 시스템, 특수한 데이터 소스에 연결할 때 필요하다.

예시: 회사 Jira에서 내 담당 이슈를 가져오는 MCP 서버.

### 1단계: 프로젝트 초기화

```bash
mkdir my-jira-mcp
cd my-jira-mcp
npm init -y
npm install @modelcontextprotocol/sdk
npm install -D typescript tsx @types/node
```

`tsconfig.json`:
```json
{
  "compilerOptions": {
    "target": "ES2022",
    "module": "Node16",
    "moduleResolution": "Node16",
    "outDir": "./dist",
    "strict": true
  }
}
```

### 2단계: 서버 코드 작성

`src/index.ts`:

```typescript
import { Server } from "@modelcontextprotocol/sdk/server/index.js";
import { StdioServerTransport } from "@modelcontextprotocol/sdk/server/stdio.js";
import {
  CallToolRequestSchema,
  ListToolsRequestSchema,
} from "@modelcontextprotocol/sdk/types.js";

const server = new Server(
  { name: "jira-mcp", version: "1.0.0" },
  { capabilities: { tools: {} } }
);

// AI에게 어떤 도구가 있는지 알려주는 핸들러
server.setRequestHandler(ListToolsRequestSchema, async () => ({
  tools: [
    {
      name: "get_my_issues",
      description: "내 담당 Jira 이슈 목록을 가져온다",
      inputSchema: {
        type: "object",
        properties: {
          status: {
            type: "string",
            enum: ["todo", "in_progress", "done"],
            description: "이슈 상태 필터",
          },
        },
      },
    },
    {
      name: "get_issue_detail",
      description: "특정 Jira 이슈의 상세 정보를 가져온다",
      inputSchema: {
        type: "object",
        properties: {
          issue_key: {
            type: "string",
            description: "이슈 키 (예: PROJ-123)",
          },
        },
        required: ["issue_key"],
      },
    },
  ],
}));

// AI가 도구를 호출할 때 실제 실행하는 핸들러
server.setRequestHandler(CallToolRequestSchema, async (request) => {
  const { name, arguments: args } = request.params;

  if (name === "get_my_issues") {
    // 실제로는 Jira API 호출
    const issues = await fetchJiraIssues(args?.status as string);
    return {
      content: [{ type: "text", text: JSON.stringify(issues, null, 2) }],
    };
  }

  if (name === "get_issue_detail") {
    const issue = await fetchJiraIssue(args?.issue_key as string);
    return {
      content: [{ type: "text", text: JSON.stringify(issue, null, 2) }],
    };
  }

  throw new Error(`알 수 없는 도구: ${name}`);
});

// Jira API 호출 함수 (실제 구현)
async function fetchJiraIssues(status?: string) {
  const JIRA_TOKEN = process.env.JIRA_TOKEN;
  const JIRA_URL = process.env.JIRA_URL;
  // fetch로 Jira REST API 호출...
  return [{ key: "PROJ-123", summary: "로그인 버그 수정", status: "in_progress" }];
}

async function fetchJiraIssue(issueKey: string) {
  // fetch로 Jira REST API 호출...
  return { key: issueKey, summary: "로그인 버그 수정", description: "..." };
}

// 서버 시작
const transport = new StdioServerTransport();
await server.connect(transport);
```

### 3단계: 빌드

```bash
npx tsc
# dist/index.js 생성됨
```

또는 빌드 없이 바로 실행:
```bash
# tsx로 TypeScript 직접 실행 (개발 중에 편함)
npx tsx src/index.ts
```

### 4단계: .mcp.json에 등록

```json
{
  "mcpServers": {
    "jira": {
      "command": "node",
      "args": ["/절대경로/my-jira-mcp/dist/index.js"],
      "env": {
        "JIRA_TOKEN": "your_jira_api_token",
        "JIRA_URL": "https://yourcompany.atlassian.net"
      }
    }
  }
}
```

tsx로 직접 실행하려면:
```json
{
  "mcpServers": {
    "jira": {
      "command": "npx",
      "args": ["tsx", "/절대경로/my-jira-mcp/src/index.ts"],
      "env": {
        "JIRA_TOKEN": "your_jira_api_token",
        "JIRA_URL": "https://yourcompany.atlassian.net"
      }
    }
  }
}
```

### 5단계: 실제로 쓰기

Claude Code를 열면 `.mcp.json`을 읽어서 서버를 자동으로 시작한다. 이후 그냥 대화하면 된다.

```
나: 내 담당 이슈 중 진행 중인 것만 보여줘.
Claude: [get_my_issues 도구 호출, status="in_progress"]
        현재 진행 중인 이슈:
        - PROJ-123: 로그인 버그 수정
        - PROJ-145: 대시보드 성능 개선

나: PROJ-123 자세히 알려줘.
Claude: [get_issue_detail 도구 호출, issue_key="PROJ-123"]
        PROJ-123: 로그인 버그 수정
        설명: OAuth 리다이렉트 후 세션이 끊기는 문제...
```

Claude가 어떤 도구를 호출하는지는 채팅창 옆에 표시된다. 별도로 명령하지 않아도 Claude가 대화 맥락을 보고 알아서 적절한 도구를 선택한다.

## 실습: filesystem MCP 연결해보기

가장 빠르게 시작할 수 있는 방법.

1. 프로젝트 루트에 `.mcp.json` 생성:

```json
{
  "mcpServers": {
    "filesystem": {
      "command": "npx",
      "args": [
        "-y",
        "@modelcontextprotocol/server-filesystem",
        "./src"
      ]
    }
  }
}
```

2. Claude Code 재시작

3. 이렇게 물어보기:
```
src 폴더 아래 파일 구조 보여줘.
```

파일 목록이 바로 나오면 성공이다.

다음 편에서는 MCP와 방향이 다른 Function Calling을 다룬다. AI가 내 코드의 함수를 선택해서 호출하는 구조 — 챗봇이나 AI 기능을 앱에 직접 내장할 때 쓰는 방법이다.
