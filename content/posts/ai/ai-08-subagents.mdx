에이전트 하나가 모든 걸 처리하면 컨텍스트가 길어지고 느려진다. 복잡한 작업은 여러 에이전트에게 나눠 맡기는 게 낫다.

---

## 멀티에이전트 구조

기본 패턴은 오케스트레이터 + 서브에이전트다.

```
[오케스트레이터]
    ├── [서브에이전트 A] — 코드 분석 담당
    ├── [서브에이전트 B] — 테스트 작성 담당
    └── [서브에이전트 C] — 문서화 담당
```

오케스트레이터는 전체 목표를 받고, 작업을 쪼개서 서브에이전트에게 분배한다. 서브에이전트는 자기 담당 작업만 처리하고 결과를 오케스트레이터에게 반환한다.

---

## 왜 이렇게 하는가

**컨텍스트 분리:** 한 에이전트가 모든 컨텍스트를 갖고 있으면 입력이 길어져서 성능이 떨어진다. 각 서브에이전트는 자기 작업에 필요한 컨텍스트만 갖는다.

**병렬 처리:** 독립적인 작업은 동시에 실행할 수 있다. 코드 분석과 문서 초안 작성이 동시에 돌아간다.

**전문화:** 각 서브에이전트에게 다른 시스템 프롬프트를 줄 수 있다. 보안 검토 전문 에이전트, 성능 최적화 전문 에이전트처럼.

**실패 격리:** 서브에이전트 하나가 실패해도 전체가 중단되지 않는다.

---

## Claude Code의 서브에이전트

Claude Code는 `Task` 도구로 서브에이전트를 실행할 수 있다. 에이전트 안에서 별도 에이전트를 생성해 독립적인 작업을 위임한다.

예를 들어 코드 리뷰 에이전트를 만들면:

```
[오케스트레이터 에이전트]
"이 PR의 변경사항을 리뷰해줘"

  → [서브에이전트 1] "보안 취약점만 찾아서 보고해"
       변경된 파일 읽기 → 보안 이슈 목록 반환

  → [서브에이전트 2] "성능 문제만 찾아서 보고해"
       변경된 파일 읽기 → 성능 이슈 목록 반환

  → [서브에이전트 3] "타입 안전성만 확인해"
       변경된 파일 읽기 → 타입 이슈 목록 반환

오케스트레이터: 세 결과를 종합해서 리뷰 리포트 생성
```

세 서브에이전트가 병렬로 실행되면 순차 실행 대비 3배 빠르다.

## 실전: Claude Code에서 서브에이전트 쓰기

Claude Code에서 직접 사용하는 방법:

```bash
# 에이전트 모드로 실행
claude --agent

# 또는 대화 중에 복잡한 태스크를 주면 Claude Code가 자동으로 서브에이전트 활용
```

Claude Code는 내부적으로 태스크 복잡도를 판단해서 서브에이전트를 자동으로 생성한다. 명시적으로 요청도 가능하다:

```
이 작업을 병렬로 처리해줘:
1. src/api/ 디렉터리의 모든 엔드포인트에 에러 처리 추가
2. 각 엔드포인트에 대한 테스트 작성
3. API 문서 업데이트

1번, 2번, 3번을 동시에 진행해서 빠르게 끝내줘.
```

## 코드로 멀티에이전트 만들기

LangGraph로 오케스트레이터-서브에이전트 패턴을 구현하는 예시:

```python
from langgraph.graph import StateGraph, END
from typing import TypedDict

class ReviewState(TypedDict):
    code: str
    security_issues: list[str]
    performance_issues: list[str]
    type_issues: list[str]
    final_report: str

# 각 서브에이전트 함수
def security_agent(state: ReviewState) -> ReviewState:
    # 보안 이슈만 분석
    issues = analyze_security(state["code"])
    return {**state, "security_issues": issues}

def performance_agent(state: ReviewState) -> ReviewState:
    # 성능 이슈만 분석
    issues = analyze_performance(state["code"])
    return {**state, "performance_issues": issues}

def orchestrator(state: ReviewState) -> ReviewState:
    # 전체 결과 종합
    report = compile_report(
        state["security_issues"],
        state["performance_issues"],
        state["type_issues"]
    )
    return {**state, "final_report": report}

# 그래프 구성
graph = StateGraph(ReviewState)
graph.add_node("security", security_agent)
graph.add_node("performance", performance_agent)
graph.add_node("orchestrator", orchestrator)

# 병렬 실행 설정
graph.add_edge("security", "orchestrator")
graph.add_edge("performance", "orchestrator")

chain = graph.compile()
```

## 언제 멀티에이전트가 필요한가

오버엔지니어링이 되기 쉬운 영역이다. 판단 기준:

**멀티에이전트가 유리한 경우:**

- 작업이 명확하게 독립적인 부분으로 나뉠 때
- 각 부분이 충분히 복잡해서 전문화가 의미 있을 때
- 병렬 처리로 시간을 아낄 수 있을 때

**단일 에이전트로 충분한 경우:**

- 작업이 순차적이고 각 단계가 이전에 의존할 때
- 전체 컨텍스트를 항상 알아야 할 때
- 태스크가 단순해서 분리 오버헤드가 이득보다 클 때
