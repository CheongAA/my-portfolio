MCP가 AI에게 도구를 연결하는 방법이라면, Function Calling은 반대 방향이다. **내 코드의 함수를 AI가 선택하고 호출하게 만드는 구조**다.

---

## 먼저, 일반 AI 응답의 한계

날씨 앱을 만든다고 하자. 사용자가 "서울 날씨 알려줘"라고 입력하면, 내 앱이 날씨 API를 호출해서 결과를 보여줘야 한다.

AI 없이는 이렇게 한다:

```
사용자 입력 파싱 → "서울"이라는 도시 추출 → 날씨 API 호출 → 결과 표시
```

AI를 써서 자연어를 처리하려면 문제가 생긴다.

```python
response = ai.chat("서울 날씨 알려줘")
print(response)
# "서울의 현재 날씨는 맑고 기온은 22도입니다. 체감 온도는..."
```

AI가 텍스트로 답했다. 이 텍스트에서 "22도"를 파싱해서 내 앱에서 써야 한다. 사용자가 "서울 날씨 어때?"라고 하면 파싱이 다르게 동작할 수 있다. 깨지기 쉽다.

---

## Function Calling의 핵심 아이디어

AI에게 "텍스트로 답하지 말고, 어떤 함수를 어떤 인자로 호출해야 하는지를 알려줘"라고 하는 것이다.

결과가 이렇게 바뀐다:

```
사용자: "서울 날씨 알려줘"

일반 AI 응답: "서울의 현재 날씨는 맑고..."

Function Calling 응답:
{
  "function": "get_weather",
  "arguments": { "city": "서울", "unit": "celsius" }
}
```

AI는 텍스트 대신 **함수 이름과 인자**를 반환한다. 이걸 받아서 내 코드가 실제 함수를 실행한다.

---

## 전체 흐름 — 이게 핵심이다

Function Calling은 한 번 주고받는 게 아니다. **왕복이 두 번** 있다.

```
1단계: 나 → AI ("서울 날씨 알려줘" + 사용할 수 있는 함수 목록)
2단계: AI → 나 (함수 이름 + 인자 반환)
3단계: 나 → 실제 날씨 API 호출 (내 코드가 실행)
4단계: 나 → AI (함수 실행 결과를 AI에게 전달)
5단계: AI → 나 (결과를 바탕으로 최종 답변 생성)
```

3단계에서 **내 코드가 실제 함수를 실행한다.** AI가 직접 실행하는 게 아니다. AI는 "이 함수를 이 인자로 호출하면 돼"라고 알려주는 역할만 한다.

4단계에서 결과를 다시 AI에게 보내야 AI가 "서울 날씨는 맑고 22도입니다"라고 최종 답변을 만들 수 있다.

이 흐름이 없으면 AI는 날씨 결과를 모른 채로 대화가 끝난다.

## 코드로 보는 전체 흐름 (OpenAI)

```python
import json
from openai import OpenAI

client = OpenAI()

# 실제 날씨 API 호출 함수 (내 코드)
def get_weather(city: str, unit: str = "celsius") -> dict:
    # 실제로는 날씨 API 호출
    return {"city": city, "temp": 22, "condition": "맑음", "unit": unit}

# AI에게 알려줄 함수 목록 (함수 스펙만, 실제 코드 아님)
tools = [
    {
        "type": "function",
        "function": {
            "name": "get_weather",
            "description": "특정 도시의 현재 날씨를 가져온다",
            "parameters": {
                "type": "object",
                "properties": {
                    "city": {
                        "type": "string",
                        "description": "날씨를 조회할 도시 이름"
                    },
                    "unit": {
                        "type": "string",
                        "enum": ["celsius", "fahrenheit"],
                        "description": "온도 단위. 기본값은 celsius"
                    }
                },
                "required": ["city"]
            }
        }
    }
]

messages = [{"role": "user", "content": "서울 날씨 알려줘"}]

# ── 1단계: AI에게 질문 + 함수 목록 전달 ──
response = client.chat.completions.create(
    model="gpt-4o",
    messages=messages,
    tools=tools
)

# ── 2단계: AI가 함수 호출을 결정 ──
assistant_message = response.choices[0].message
print(assistant_message.tool_calls[0].function.name)       # "get_weather"
print(assistant_message.tool_calls[0].function.arguments)  # '{"city": "서울"}'

# ── 3단계: 내 코드가 실제 함수 실행 ──
tool_call = assistant_message.tool_calls[0]
args = json.loads(tool_call.function.arguments)
result = get_weather(**args)  # {"city": "서울", "temp": 22, "condition": "맑음"}

# ── 4단계: 실행 결과를 AI에게 다시 전달 ──
messages.append(assistant_message)  # AI 응답 추가
messages.append({
    "role": "tool",
    "tool_call_id": tool_call.id,
    "content": json.dumps(result)   # 함수 실행 결과
})

# ── 5단계: AI가 최종 답변 생성 ──
final_response = client.chat.completions.create(
    model="gpt-4o",
    messages=messages,
    tools=tools
)

print(final_response.choices[0].message.content)
# "서울의 현재 날씨는 맑고 기온은 22도입니다."
```

핵심은 **messages 배열에 계속 추가**하는 것이다. AI 응답 → 함수 결과 → 다시 AI 호출. 이 흐름이 맞아야 동작한다.

## 코드로 보는 전체 흐름 (Claude)

Anthropic은 같은 개념을 "Tool Use"라고 부른다.

```python
import anthropic
import json

client = anthropic.Anthropic()

def get_weather(city: str) -> dict:
    return {"city": city, "temp": 22, "condition": "맑음"}

tools = [
    {
        "name": "get_weather",
        "description": "특정 도시의 현재 날씨를 가져온다",
        "input_schema": {
            "type": "object",
            "properties": {
                "city": {
                    "type": "string",
                    "description": "날씨를 조회할 도시 이름"
                }
            },
            "required": ["city"]
        }
    }
]

messages = [{"role": "user", "content": "서울 날씨 알려줘"}]

# ── 1단계 + 2단계 ──
response = client.messages.create(
    model="claude-opus-4-6",
    max_tokens=1024,
    tools=tools,
    messages=messages
)

# ── 3단계: 함수 실행 ──
tool_use_block = next(b for b in response.content if b.type == "tool_use")
result = get_weather(**tool_use_block.input)

# ── 4단계: 결과를 AI에게 전달 ──
messages.append({"role": "assistant", "content": response.content})
messages.append({
    "role": "user",
    "content": [
        {
            "type": "tool_result",
            "tool_use_id": tool_use_block.id,
            "content": json.dumps(result)
        }
    ]
})

# ── 5단계: 최종 답변 ──
final_response = client.messages.create(
    model="claude-opus-4-6",
    max_tokens=1024,
    tools=tools,
    messages=messages
)

print(final_response.content[0].text)
# "서울의 현재 날씨는 맑고 22도입니다."
```

## AI가 함수를 선택하는 기준

여러 함수를 등록하면 AI가 사용자 의도에 맞는 것을 고른다.

```python
tools = [
    {"name": "get_weather",         "description": "날씨 조회"},
    {"name": "create_calendar_event","description": "캘린더에 일정 추가"},
    {"name": "send_slack_message",  "description": "슬랙 메시지 전송"},
]
```

- "서울 날씨 알려줘" → `get_weather` 선택
- "내일 오전 10시 미팅 잡아줘" → `create_calendar_event` 선택
- "팀 채널에 배포 완료 알려줘" → `send_slack_message` 선택
- "미팅 잡고 슬랙에도 알려줘" → 두 함수를 순서대로 호출

AI가 `description`을 읽고 판단한다. **description을 명확하게 쓸수록 정확하게 선택한다.**

---

## AI가 함수를 호출하지 않을 수도 있다

AI가 항상 함수를 호출하는 건 아니다. tools를 넘겨도 일반 텍스트로 답할 수 있다.

```
사용자: "안녕"
→ AI: "안녕하세요!" (함수 호출 없음)

사용자: "서울 날씨 알려줘"
→ AI: get_weather 호출
```

함수 호출 여부를 체크하는 코드가 필요하다:

```python
# OpenAI
if response.choices[0].message.tool_calls:
    # 함수 호출 처리
else:
    # 일반 텍스트 응답 처리
    print(response.choices[0].message.content)

# Claude
if response.stop_reason == "tool_use":
    # 함수 호출 처리
else:
    # 일반 텍스트 응답 처리
    print(response.content[0].text)
```

## 실습: 할 일 챗봇 만들기

처음부터 끝까지 동작하는 예시.

```python
import json
from openai import OpenAI

client = OpenAI()

# 실제 할 일 저장소
todos = []

# 실제 함수들
def add_todo(title: str, priority: str = "medium") -> dict:
    todo = {"id": len(todos) + 1, "title": title, "priority": priority, "done": False}
    todos.append(todo)
    return {"success": True, "todo": todo}

def list_todos() -> dict:
    return {"todos": todos}

def complete_todo(id: int) -> dict:
    for todo in todos:
        if todo["id"] == id:
            todo["done"] = True
            return {"success": True}
    return {"success": False, "error": "찾을 수 없음"}

# AI에게 알려줄 함수 스펙
tools = [
    {
        "type": "function",
        "function": {
            "name": "add_todo",
            "description": "할 일 목록에 새 항목을 추가한다",
            "parameters": {
                "type": "object",
                "properties": {
                    "title": {"type": "string", "description": "할 일 제목"},
                    "priority": {"type": "string", "enum": ["low", "medium", "high"]}
                },
                "required": ["title"]
            }
        }
    },
    {
        "type": "function",
        "function": {
            "name": "list_todos",
            "description": "할 일 목록을 보여준다",
            "parameters": {"type": "object", "properties": {}}
        }
    },
    {
        "type": "function",
        "function": {
            "name": "complete_todo",
            "description": "할 일을 완료 처리한다",
            "parameters": {
                "type": "object",
                "properties": {
                    "id": {"type": "integer", "description": "완료할 할 일의 ID"}
                },
                "required": ["id"]
            }
        }
    }
]

# 함수 실행 매핑
function_map = {
    "add_todo": add_todo,
    "list_todos": list_todos,
    "complete_todo": complete_todo,
}

def chat(user_input: str, messages: list) -> str:
    messages.append({"role": "user", "content": user_input})

    response = client.chat.completions.create(
        model="gpt-4o", messages=messages, tools=tools
    )
    assistant_message = response.choices[0].message

    # 함수 호출이 없으면 그냥 반환
    if not assistant_message.tool_calls:
        messages.append(assistant_message)
        return assistant_message.content

    # 함수 실행
    messages.append(assistant_message)
    for tool_call in assistant_message.tool_calls:
        fn = function_map[tool_call.function.name]
        args = json.loads(tool_call.function.arguments)
        result = fn(**args)

        messages.append({
            "role": "tool",
            "tool_call_id": tool_call.id,
            "content": json.dumps(result, ensure_ascii=False)
        })

    # 결과 반영해서 최종 답변
    final = client.chat.completions.create(
        model="gpt-4o", messages=messages, tools=tools
    )
    final_message = final.choices[0].message
    messages.append(final_message)
    return final_message.content


# 실행 예시
messages = []
print(chat("PR 리뷰 high priority로 추가해줘", messages))
# "PR 리뷰를 높은 우선순위로 추가했습니다."

print(chat("지금 할 일 목록 보여줘", messages))
# "현재 할 일 목록: 1. PR 리뷰 (high)"

print(chat("1번 완료 처리해줘", messages))
# "PR 리뷰를 완료 처리했습니다."
```

## 정리

Function Calling의 흐름을 한 줄로:

> AI가 "이 함수를 이 인자로 호출해" → 내 코드가 실제 실행 → 결과를 AI에게 전달 → AI가 최종 답변

AI는 판단만 한다. 실행은 내 코드가 한다. 결과를 다시 AI에게 넘겨야 대화가 완성된다.

---

다음 편에서는 Function Calling의 사촌 격인 Structured Output을 다룬다. 함수를 호출하는 게 아니라, AI 응답 자체를 내가 원하는 JSON 형태로 강제하는 방법이다.
