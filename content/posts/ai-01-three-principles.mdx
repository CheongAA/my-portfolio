AI를 쓰는데 자꾸 이런 상황이 반복된다면.

- 답변이 원하는 방향과 다르다. 다시 물어보면 또 다르다. 몇 번을 주고받다 결국 내가 직접 고친다.
- 유료 플랜인데 한도가 금방 찬다.
- 매 대화마다 "이 프로젝트는 Next.js야, TypeScript 써" 같은 설명을 반복한다.

세 가지 다 다른 문제처럼 보이지만 원인은 하나다. **AI에게 뭘, 어떻게 보내는지**를 신경 쓰지 않은 것.

---

## 원칙 1. 관련된 것만 보내기

가장 흔한 실수 — 에러가 났을 때 파일 전체를 복사해서 보내는 것.

500줄짜리 파일에서 실제 문제가 되는 코드는 20줄이다. 나머지 480줄은 AI가 읽어야 하는 노이즈다. AI의 주의가 분산되고, 쓸모없는 토큰이 쌓인다.

**지워도 되는 것:**
- 에러와 관련 없는 다른 함수들
- 무관한 import 목록
- 이미 동작하는 부분

**남겨야 하는 것:**
- 에러가 나는 함수
- 그 함수가 쓰는 타입 / 인터페이스
- 에러 메시지 전체

before / after:

```
❌ [파일 전체 500줄]
왜 안 되죠?
```

```
✅ Next.js App Router에서 서버 컴포넌트 안에 useEffect를 썼더니
"You're importing a component that needs useState" 에러가 납니다.

관련 코드:
[에러 나는 컴포넌트 15줄]
```

첫 번째는 7번 주고받았다. 두 번째는 1번에 해결됐다.

## 원칙 2. 원하는 형식을 미리 말하기

AI는 형식을 지정하지 않으면 알아서 길게 쓴다. 원인 설명, 대안 두 가지, 장단점 비교, 추가 고려사항까지. 그 중에 내가 필요한 건 수정된 코드 10줄인데.

원하는 게 뭔지 미리 말하면 그대로 나온다.

| 원하는 것 | 추가할 말 |
|-----------|-----------|
| 코드만 | "코드만 줘, 설명 없이" |
| 짧은 요약 | "3줄 이내로 요약해줘" |
| 특정 형식 | "TypeScript로", "함수형으로", "주석 없이" |
| 여러 선택지 | "3가지 방법 각각 한 줄씩" |

```
❌ fetchUserData 함수에서 타입 에러가 납니다. 고쳐주세요.

✅ fetchUserData 함수 타입 에러 수정해줘. 수정된 함수 코드만.
```

두 번째는 수정된 함수 하나만 돌아온다. 바로 붙여넣고 확인 끝.

## 원칙 3. 반복 설명을 세팅으로 고정하기

매 대화 첫 메시지에 이런 걸 쓰고 있다면:

```
이 프로젝트는 Next.js 15 App Router야. TypeScript strict mode 쓰고,
Tailwind CSS v4로 스타일링해. 서버 컴포넌트 우선으로 설계해줘.
```

하루에 대화를 10번 한다면 같은 설명을 10번 보내는 거다.

이건 시스템 프롬프트나 프로젝트 설정으로 한 번만 세팅하면 된다. Claude Projects, ChatGPT Projects 모두 이 기능을 지원한다. 세팅해두면 그 안에서 여는 모든 대화에 자동 적용된다.

이 방법은 다음 편에서 구체적으로 다룬다.

## 실습

지금 작업 중인 코드에서 에러가 하나 있다면:

1. 에러가 나는 부분만 잘라서 30줄 이내로 줄인다
2. 에러 메시지 전체를 복사한다
3. "무엇을 하려 했는지 + 에러 메시지 + 코드 + 원하는 답변 형식" 순서로 요청을 다시 쓴다

코드를 줄이면 답이 나빠질 것 같지만 실제로는 반대다. 핵심만 남기면 AI가 핵심에 집중한다.
